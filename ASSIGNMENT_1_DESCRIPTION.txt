Adaptive Treatment Recommendation System using MultiArmed Bandit Learning
Problem Scenario
You are a data scientist working in cancer research in collaboration with a medical institution
conducting clinical trials for different treatment protocols. A hospital is evaluating multiple
medicines for treating a chronic disease, where patient responses vary due to hidden factors such
as age, immunity level, genetic background, and disease severity. Since the most effective
treatment is not known at the beginning, the goal is to develop an intelligent recommendation
system that learns from patient outcomes over time and progressively identifies the optimal
medicine for future patients.
Each medicine is treated as an arm in a Multi-Armed Bandit (MAB) problem.
Each group must create a custom synthetic patient-treatment dataset based on the group
number



1. Dataset Creation Rules
Let group number = G.
Use: random.seed(G),numpy.random.seed(G) to ensure reproducibility.

𝑃𝑖 = 0.4 + ((𝐺 + 𝑖)𝑚𝑜𝑑6) × 0.07

PART #2 - DP

Autonomous Drone Rescue Using Dynamic Programming
Problem Statement
Design and implement a reinforcement learning agent using dynamic programming - Value Iteration or
Policy Iteration to compute an optimal policy for an autonomous rescue drone operating in a disaster zone.
The drone must learn how to rescue stranded civilians while managing battery power, avoiding dangerous
zones, and reaching charging stations when necessary. The problem must be modelled as a finite Markov
Decision Process (MDP). The student will:
● Implement a custom Drone Rescue environment.
● Use dynamic programming to compute the optimal value function and policy.
● Analyse how state design and reward design affect convergence and rescue efficiency.
Scenario
A rescue drone is deployed in a disaster-hit city after an earthquake. The city is represented as a grid
world. The drone must:
● rescue stranded civilians,
● avoid dangerous areas,
● manage battery usage,
● and reach charging stations before battery depletion.
The environment contains:
● Safe zones
● Danger zones
● Charging stations
● Rescue targets
● Wind zones that disturb movement
The drone begins with limited battery power and must maximize total rescue reward.
Environment Description:
Grid Size



● If the group ID ends with 0–4:
● Use a 5×5 grid.
● If the group ID ends with 5–9:
● Use a 6×6 grid.
Initial Environment Configuration
The placement of rescue targets, charging stations, danger zones, and blocked cells must depend on the
group ID.
If the group ID ends with 0–4:
● Use:
● 2 rescue targets
● 1 charging station
● 3 danger zones
● 2 blocked cells
If the group ID ends with 5–9:
● Use:
● 3 rescue targets
● 2 charging stations
● 4 danger zones
● 3 blocked cells
Students must clearly explain:
● the grid configuration,
● symbol placement,
● and starting battery level.
Cell Types
Symbol Meaning
S Start position
F Free/Safe cell
D Dangerous zone
R Rescue target
C Charging station
W Wind zone
X Blocked cell / obstacle
Drone Behaviour
The drone:
● starts at S, The starting position (S) must be fixed at the top-left corner of the grid.
● has limited battery,
● loses battery every move,
● gains reward by rescuing civilians,
● and must avoid running out of power.
Actions
At each state, the drone may:
● Move Up
● Move Down
● Move Left
● Move Right
● Hover (remain in the same cell)
Hover Action
● If the drone is on a charging station (C):
● hovering increases battery by +2 units
● battery cannot exceed maximum capacity.
● If the drone is not on a charging station:
● hovering consumes 1 battery unit like a normal move.
Special Environment Rules
1. Battery System
Each action consumes 1 battery unit.
The maximum battery capacity depends on the group ID:
● If the group ID ends with an even digit:
● maximum battery = 10 units
● If the group ID ends with an odd digit:
● maximum battery = 15 units
If battery becomes 0: the episode terminates immediately.
2. Charging Stations (C)
When the drone enters a charging station: battery becomes full again.
3. Rescue Targets (R)
When the drone reaches a rescue cell:
● reward is obtained,
● the rescue target disappears,
● the cell becomes a free cell (F).
4. Dangerous Zones (D)
Danger cells represent fire, unstable buildings, or radiation.
Entering a danger zone:
● gives heavy negative reward,
● but does not terminate the episode immediately.
5. Wind Zones (W)
Wind cells introduce stochastic transitions.
If the drone is currently on a wind cell (W) and attempts a movement action: there is a probability that the
movement direction changes randomly.
If wind disturbance occurs: the actual movement direction must be selected uniformly from:
● Up,
● Down,
● Left,
● Right.
The wind probability depends on the group ID:
● 20% if the group ID ends with 0–4
● 30% if the group ID ends with 5–9
The student must clearly explain the transition dynamics used.
6. Blocked Cells (X)
Blocked cells represent inaccessible regions such as collapsed buildings or debris.
● The drone cannot enter blocked cells.
● If an action would move the drone into a blocked cell:
● the drone remains in its current position,
● the action still consumes 1 battery unit.
Episode Termination
An episode ends when:
● Battery becomes zero
● All rescue targets are rescued
● Maximum step limit exceeded: 50 for 5×5 grids and 75 for 6×6 grids.
Environment Dynamics:
1. State Space
Each state should minimally contain:
● Drone position (row, col)
● Current battery level
● Status of each rescue target (rescued / not rescued)
The student must:
● clearly explain the state representation.
2. Action Space
Possible actions: Up, Down , Left , Right, Hover
The student must implement a function that returns all valid actions available from a given state. The
student must ensure that only valid actions are allowed at each state.
3. Rewards
The environment must use the following reward structure.
Event Reward
Rescue target reached 20
Enter danger zone -10
Battery exhausted -20
Reach charging station 5
Regular movement -1



Expected Outcomes
1. Custom Drone Rescue Environment – 1 mark
Implement a custom environment with: reset() , step(action) and render(). The environment must
correctly handle: battery updates, rescue target removal, stochastic wind movement, charging stations,
blocked cells and reward computation.
2. Dynamic Programming Solution – 2 mark
Using Value Iteration or Policy Iteration:
1. Enumerate reachable states.
2. Compute:
● optimal value function V∗(s)
● optimal policy π∗(s)
3. Use stopping threshold:
● θ=10−3
4. Show:
● convergence iterations,
● runtime,
● final delta/error.
3. Policy Visualisation – 1 mark
Visualise: drone movement directions, rescue sequence, preferred charging behaviour, avoidance of
dangerous zones.
Students may use: arrows, heat maps, grid overlays, or animated trajectories.
4. State-Value Analysis – 1 mark
Choose a meaningful slice of the state space.
For example:
● Fix rescue target locations and battery level,
● Vary only drone position,
● Plot heatmap of V∗(s).
Students must explain observed patterns.
5. DP Scalability Discussion – 1 mark
Explain: Curse of Dimensionality
How state space changes if:
● grid becomes 10×10,
● more rescue targets are added,
● weather conditions become dynamic.
Is DP sufficient? Discuss:
● why DP becomes difficult,
● how Deep RL methods could help,
● relation to real-world autonomous drone systems.
